# Code Bagu 外行人手把手教程

> 你不需要懂代码。你只需要会检查"作业有没有漏写"。

---

## 前置准备

1. **Kimi Code CLI** — 去 kimi.moonshot.cn 下载安装
2. **DeepSeek** — 网页版或 API 都行（用来预处理需求）
3. **一个想做的新项目** — 什么都行，小程序、脚本、工具

---

## 第一步：配置 Kimi Code 的 Skill（一次性）

### 1.1 打开 Kimi Code 的设置

找到 **Skill 管理** 或 **System Prompt 设置**。

### 1.2 复制粘贴 Skill 文件

打开本仓库的 `code_bagu_skill.md`，**全文复制**，粘贴进 Kimi Code 的 Skill 输入框。

### 1.3 保存

点保存。以后每次 Kimi Code 写代码，都会自动按八股格式输出。

---

## 第二步：项目根目录放 AGENTS.md（每个项目一次）

### 2.1 复制模板

复制本仓库的 `examples/AGENTS_example.md`，放到你项目的根目录。

### 2.2 改三处

```markdown
# AGENTS.md — [你的项目名]

> 本文件约束 `~/[你的项目路径]/` 目录及全部子目录。

## 项目上下文

- **目标平台**：[你的运行环境]
- **前端**：[用什么做界面]
- **后端**：[用什么做服务]
```

别的不用改。这个文件告诉 AI："在这个项目里，按八股规范写代码。"

---

## 第三步：双引擎流水线（每次写功能都用）

### 3.1 第一步：用 DeepSeek 预处理需求

把你的人话需求丢给 DeepSeek，让它拆成结构化指令。

**你的输入（人话）：**

```
我想做一个跳绳计数的小程序，摄像头拍到人跳一下就加一。
```

**DeepSeek 的输出（结构化）：**

```markdown
## 需求拆解

### 做什么
- 通过摄像头捕捉人体骨骼点
- 检测手腕/肩膀的 Y 轴波动
- 每完成一次起跳→腾空→落地周期，计数 +1

### 不做什么
- 不做姿态矫正（弯腰/直腿不管）
- 不做双摇识别（两次绕绳算一次）
- 不做倒跳过滤

### 边界条件
- 置信度阈值：骨骼点置信度 > 0.8 才计入
- 最小间隔：两次计数间隔 ≥ 0.3 秒
- 降级路径：摄像头丢失 → 提示用户调整位置

### 输入输出
- 输入：摄像头帧（实时视频流）
- 输出：int 计数 + float 完成度百分比
```

### 3.2 第二步：把结构化指令丢给 Kimi Code

**你的指令：**

```
按 Code Bagu 八股规范，先输出完整框架，不要写实现代码。

需求：
[粘贴 DeepSeek 的结构化输出]

要求：
1. 先输出文件级拓扑图头
2. 再输出函数级八股框架（破题/承题/起讲/入手/起股/中股/后股/束股）
3. 最后输出 AI 自检清单

我检查通过后，你再填充实现。
```

### 3.3 第三步：你检查结构（关键！不懂代码也能做）

Kimi Code 会输出这样的框架：

```python
# 破题：基于肩膀/臀部 Y 轴波动的振幅法，检测跳绳完整周期；不做姿态矫正与双摇识别。
# 承题：依赖 kps 关键点字典、滑动窗口、全局极值。
# [起讲] 选择信号源 → 更新滑动窗口 → 计算振幅 → 相位状态机判定 → 满足条件则计数。
def count_jumps():
    # ==== 起股 ====
    # 取：kps（关键点字典）
    # 验：肩膀/髋部均不可见 → 返回无效
    
    # ==== 中股 ====
    # 算：滑动窗口归一化、参数映射
    # 转：原始坐标 → 归一化百分比
    
    # ==== 后股 ====
    # ✓ 正路径：满足最小腾空时间 + 最小间隔 → 计数
    # ✗ 降级路径：条件不满足 → 仅复位状态，不计数
    
    # ==== 束股 ====
    # 给出：(triggered, per, upper)
    # 留下：更新状态机、滑动窗口
```

**你检查什么（不用懂代码）：**

| 检查项 | 看什么 | 合格标准 |
|--------|--------|---------|
| 破题 | 第二句 | 必须以"不做"开头，明确范围边界 |
| 起股 | 验 | 有没有写验证条件？漏了 = 可能崩 |
| 后股 | ✗ 降级路径 | 必须有，哪怕是"N/A" |
| 束股 | 留下 | 有没有副作用声明？漏了 = 可能有坑 |
| 对偶 | 🔒/🔓、✓/✗ | 成对出现，数量平衡 |

**全部打勾 → 说"通过，填代码"**  
**有漏的 → 指出具体哪一段，让 AI 补**

### 3.4 第四步：AI 填代码，你再微调

代码填完后，你不需要读每一行。只需要：

1. 运行测试
2. 如果出 bug，让 AI 定位到具体"股"

**不要说：**"这里好像不对，你改改。"  
**要说：**"后股的降级路径没有处理网络中断，补一个。"

## 第四步：固化你的黄金模板

某个模块跑通后，把它的八股框架抽出来，存成模板。

### 4.1 模板格式

```markdown
## [模块名] — 黄金八股模板

```python
# 破题：[做什么]；不做[什么]。
# 依赖：[用到的工具/库]

def [函数名]():
    # ==== 起股 ====
    # 取：[输入]
    # 验：[验证条件]
    
    # ==== 中股 ====
    # 算：[核心计算]
    # 转：[状态转换]
    
    # ==== 后股 ====
    # ✓ 正路径：[正常流程]
    # ✗ 降级路径：[异常处理]
    
    # ==== 束股 ====
    # 给出：[返回值]
    # 留下：[副作用]
```

### 4.2 下次用

```
按[模块名]的黄金八股模板写，阈值不许改，降级路径不许少。
```

## 常见错误 & 修正

### 错误 1：AI 直接写代码，不出框架

**原因：** 你的指令没有明确"先出框架"。  
**修正：** 加一句"不要写实现。先按八股规范输出完整框架。"

### 错误 2：AI 漏写降级路径

**原因：** AI 偷懒，后股只写正路径。  
**修正：** 检查清单里强制"后股必须含 ✓ 正路径 + ✗ 降级路径"。

### 错误 3：上下文爆了（256K 不够）

**原因：** 每个函数都写完整八股，文件太多。  
**修正：** 用"战略性分级"——核心函数写完整八股，工具函数写极简八股。

### 错误 4：AI 格式不统一

**原因：** AI 有时候写 `# 起股：` 有时候写 `# ==== 起股 ====`。  
**修正：** AGENTS.md 里写明"股标记统一为 `==== X股 ====`"，AI 自检清单里有格式检查。

## 一句话总结

**八股编码 = 把"事后验收"变成"事前检查"。**  
你不懂代码，但能看懂"有没有降级路径"——这个结构检查替代了 5-8 遍的现场测试。
