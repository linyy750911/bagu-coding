# 行为准则

## 我们的承诺

这是一个包容的社区。我们相信：

- 非程序员也能参与软件工程的质量把控
- 代码审查不应该只属于会写代码的人
- 结构化的思维比具体的语法知识更通用

## 我们的标准

### 鼓励的行为

- 用代码八股模板成功降低了 bug 迭代次数，来分享
- 发现了更好的结构化注释格式，来讨论
- 把八股编码适配到了新的 AI 工具，来投稿
- 帮助新成员理解"先检查结构，再填代码"的流程

### 禁止的行为

- 嘲讽他人"不懂代码"
- 发表任何形式的歧视性言论
- 恶意灌水或刷 Issue
- 未经允许公开他人隐私信息

## 适用范围

本行为准则适用于所有项目空间（Issue、PR、Discussion）以及以项目名义进行的公共交流。

## 执行

如果你发现不当行为，请在 Issue 中 @maintainer 或联系项目维护者。所有报告将被保密处理。
