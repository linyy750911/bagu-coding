# 贡献指南

## 投稿黄金模板（最受欢迎）

如果你用了代码八股之后发现某个模块的改 bug 次数大幅减少，把它的八股框架抽出来就是黄金模板！

### 投稿格式

```
## [模块名] — 黄金八股模板

- **来源项目**：[比如"智慧体育"]
- **适用场景**：[一句话描述这模板用在什么类型的问题上]
- **效果对比**：改X遍 → 改Y遍
- **依赖库**：[列出]

```python
# 破题：xxx；不做xxx。
# 依赖：xxx

def xxx():
    # ==== 起股 ====
    # 取：xxx
    # 验：xxx
    
    # ==== 中股 ====
    # 算：xxx
    # 转：xxx
    
    # ==== 后股 ====
    # ✓ 正路径：xxx
    # ✗ 降级路径：xxx
    
    # ==== 束股 ====
    # 给出：xxx
    # 留下：xxx
```
```

### PR 要求

- 模板文件放到 `examples/` 目录
- 命名：`golden_template_<用途>.py` 或 `golden_template_<用途>.md`
- PR 描述里写"用八股前改 X 遍，用八股后改 Y 遍"

---

## 提交改进建议

1. **Bug 报告**：开 Issue，描述预期行为和实际行为
2. **文档改进**：发现哪里写得不清楚，直接开 Issue
3. **新平台适配**：比如你让八股编码跑在 Cursor/Windsurf/Copilot 上，欢迎分享配置方法

---

## 开发约定

- 所有代码注释用中文
- Skill 文件（`code_bagu_skill.md`）是核心，改之前先开 Issue 讨论
- 实验数据更新要附对比截图或日志

这是一个包容的社区。无论你是不是程序员，你的贡献都有价值。我们相信：**边界条件检查不是程序员的特权**。
