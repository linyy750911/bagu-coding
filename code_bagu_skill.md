# Skill: Code Bagu（代码八股）— AI 优先版

> 八股编码的核心受众是 AI，不是人类。  
> 所有结构设计的唯一目标是：**让 AI 能 100% 准确、零歧义地解析代码意图、数据流向、状态机和边界条件。**  
> 人类觉得"注释多"是特征，不是 bug。

---

## 1. 为什么 AI 需要八股

| 无八股代码 | AI 的困境 | 有八股代码 | AI 的能力 |
|-----------|----------|-----------|----------|
| 裸函数体 | 需逐行推理意图，易错 | 破题+起讲 | 直接提取契约，O(1) |
| 嵌套 if/else | 需构建 AST 才能找边界 | 起股/中股/后股/束股 | 直接定位正路径/降级路径 |
| 资源申请分散 | 难以配对申请-释放 | 入手+对偶标记 | 静态检查资源泄漏 |
| 隐式状态流转 | 需全局数据流分析 | 拓扑图头 | 直接读取数据流向 |

---

## 2. 三层递归结构（AI 解析锚点）

### 2.1 文件级 — 拓扑图头（必须）

AI 通过拓扑图头在 **10 秒内** 理解模块全貌。格式必须严格统一：

```python
"""
<模块名> — <一句话职责> (<文件名>)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
拓扑图:
  输入: <输入1> / <输入2>
  输出: <输出1> → <下游模块>

  数据流向:
    <A> → <B> → <C>
    <D> → <E> (条件分支)

修改风险点:
  ⚠️ <行号>: <改了会怎样>
  ⚠️ <行号>: <改了会怎样>

最近修改:
  YYYY-MM-DD: <修改内容>
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
"""
```

**AI 解析锚点：**
- `输入:` 后面列出的变量/参数 → AI 提取输入接口
- `输出:` 后面的箭头链 → AI 构建数据依赖图
- `⚠️` 标记 → AI 识别高敏感代码区域
- `最近修改:` → AI 理解演进历史

### 2.2 函数级 — 八股段落（必须）

每个非 trivial 函数必须有完整的八股框架。AI 通过段落标记直接跳转：

```python
# 破题：<一句话做什么>；<一句话不做什么>。
# 承题：<依赖声明>。<前置条件>。<上下文>。
# [起讲] <设计意图，禁止实现细节>
# 入手：<资源申请，必须成对出现>
def func():
    # ==== 起股 ====
    # 取：<获取输入>
    # 验：<验证合法性>
    
    # ==== 中股 ====
    # 算：<核心计算>
    # 转：<状态转换>
    
    # ==== 后股 ====
    # 正路径：<正常流程>
    # 降级路径：<异常/降级处理>
    
    # ==== 束股 ====
    # 给出：<返回什么>
    # 留下：<副作用/状态变更>
```

**AI 解析锚点：**
- `# 破题：` → AI 提取函数契约（做什么/不做什么）
- `# ==== X股 ====` → AI 直接定位到对应段落
- `# 正路径：` / `# 降级路径：` → AI 提取测试用例（正例+反例）
- `# 给出：` → AI 提取返回值语义
- `# 留下：` → AI 追踪副作用

### 2.3 块级 — 对偶标记（复杂块内）

if/for/try/while 等复杂块内部，用对偶标记帮助 AI 理解逻辑对称性：

```python
if condition:
    # ✓ 正路径：<说明>
    ...
else:
    # ✗ 降级路径：<说明>
    ...

try:
    # 🔒 申请：<资源>
    ...
finally:
    # 🔓 释放：<资源>
    ...
```

**AI 解析锚点：**
- `✓` / `✗` → AI 识别正/反路径
- `🔒` / `🔓` → AI 配对资源申请-释放
- `↗` / `↘` → AI 识别状态升降（计数器增减、标志位切换）

---

## 3. 对偶强制（AI 可静态检查）

所有逻辑必须以**成对**形式出现。AI 通过搜索对偶标记来验证完整性：

| 对偶类型 | AI 搜索模式 | 人类示例 | AI 检查逻辑 |
|----------|------------|---------|------------|
| **资源对偶** | `🔒` ↔ `🔓` | `open()` ↔ `close()` | 统计 🔒 和 🔓 数量是否相等 |
| **路径对偶** | `✓` ↔ `✗` | `if success:` ↔ `else:` | 每个 ✓ 必须有对应的 ✗ |
| **变换对偶** | `📥` ↔ `📤` | `decode()` ↔ `encode()` | 输入变换和输出变换对称 |
| **校验对偶** | `🔍` ↔ `✅` | `read()` ↔ `assert_valid()` | 任何写入前有验证标记 |
| **计数对偶** | `↗` ↔ `↘` | `count += 1` ↔ `count -= 1` | 增减操作数量平衡 |

## 4. 空八股规则（AI 版）

**人类版**认为"取：无"是噪音要删除。  
**AI 版**认为空八股是**显式声明**——AI 知道"这里本应有验证，但作者明确声明不需要"。

因此：
- **允许空八股**，但格式必须统一为 `# <股名>：N/A` 或 `# <股名>：无需`
- AI 看到 `N/A` 就知道此处被显式跳过，不会猜测是否有遗漏

```python
# ==== 起股 ====
# 取：N/A  （无外部输入，由调用方注入）
# 验：N/A  （状态机内部流转，无需额外验证）
```

## 5. 格式铁律（AI 解析的前提）

### 5.1 注释行必须以 `#` 开头，空格规范

```python
# ✅ 正确 — AI 正则 ^\s*# 可匹配
# 破题：xxx

# ❌ 错误 — AI 可能漏匹配
#破题：xxx  （无空格）
#  破题：xxx （多空格）
```

### 5.2 股标记必须严格统一

```python
# ✅ 正确
# ==== 起股 ====
# ==== 中股 ====
# ==== 后股 ====
# ==== 束股 ====

# ❌ 错误 — AI 无法统一匹配
# ==== 起股 ==== → 取：xxx
# ---- 起股 ----
# [起股]
```

### 5.3 破题必须严格两句

```python
# ✅ 正确
# 破题：获取用户订单列表；不做分页逻辑和权限校验。

# ❌ 错误 — AI 无法拆分"做什么/不做什么"
# 破题：获取用户订单列表并分页返回。
```

### 5.4 拓扑图头必须包含 5 个字段

```python
# ✅ 正确
"""
模块名 — 职责 (文件名)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
拓扑图:
  输入: xxx
  输出: yyy → zzz

  数据流向:
    A → B → C

修改风险点:
  ⚠️ 第X行: xxx

最近修改:
  2026-05-11: xxx
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
"""
```

## 6. AI 自检清单（每写完一个函数必须执行）

```
[ ] 破题：两句话，分号分隔，第二句以"不做"开头
[ ] 承题：声明依赖、前置条件
[ ] 起讲：[起讲] 标记存在，无实现细节
[ ] 入手：资源申请成对（如有）
[ ] 起股：==== 起股 ==== 标记存在
[ ] 中股：==== 中股 ==== 标记存在
[ ] 后股：==== 后股 ==== 标记存在，含正路径+降级路径
[ ] 束股：==== 束股 ==== 标记存在，含给出+留下
[ ] 对偶检查：🔒/🔓、✓/✗、↗/↘ 数量平衡
[ ] 格式检查：所有注释行以"# "开头（#后有空格）
[ ] 格式检查：股标记统一为 "==== X股 ===="（8个= + 空格 + 股名 + 空格 + 8个=）
```

---

## 7. 输出顺序

1. **先输出八股框架**（所有注释，不含实现代码）
2. **再输出实现代码**（填充到框架中）
3. **最后 AI 自检**（逐条核对清单）
4. **额外输出：AI 解析摘要**（用机器可读格式总结本函数的契约）

---

## 8. 禁止事项

- ❌ 禁止破题直抄文件名
- ❌ 禁止起讲写实现细节
- ❌ 禁止后股省略降级路径（哪怕只是 `# 降级路径：N/A`）
- ❌ 禁止单股出现（任何股必须是标记+内容，不能只写标记）
- ❌ 禁止格式不统一（全文件必须用同一套标记风格）
- ❌ 禁止用 `//` 或 `/* */` 代替 `#`（Python 代码）
